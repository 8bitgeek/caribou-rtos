29  * @brief If the CARIBOU_TIMER_F_DYNAMIC flag is set, then it indicates that the
30  * timer was dynamically allocated on the heap with caribou_timer_create(), and so will
31  * be disposed of properly by caribou_timer_delete().
32  * Otherwise, if the CARIBOU_TIMER_F_DYNAMIC is not set, then it is assumed that it is
33  * a statically allocated, or stack allocated instance of a caribou_timer_t structure,
34  * and so caribou_timer_delete() will make no attempt to de-allocate the memory associated with
35  * the timer structure.
36  */
37 #define CARIBOU_TIMER_F_DYNAMIC 0x01

39 /**
40  * @brief Normally a caribou_timer_t will trigger a timeout callback upon the termination
41  * of the timeout period, and once the callback is serviced, the period counter is then reloaded
42  * and the timer restarted, such that the timer will repeat automatically at the specified period.
43  * If the CARIBOU_TIMER_F_ONESHOT flag is set, then the timer callback is called exactly once at the
44  * first expiration of the period counter. In either case, calling caribou_timer_reset() will
45  * re-load the period counter and start the timer again.
46  */
47 #define CARIBOU_TIMER_F_ONESHOT 0x02 /// A one-shot timer, no-reload
